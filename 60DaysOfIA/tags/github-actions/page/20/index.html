Creating an Azure Static Web Resource‚Äã

Next, you‚Äôll create an Azure Static Web App resource using the Azure CLI. The Azure Static Web App resource is the container for the app and its settings.

To create it, run the following command in your terminal:

az staticwebapp create \
  --name virtual-stylist-chat \
  --resource-group <your resource group> \
  --location westus2 \
  --source virtual-stylist-chat \
  --branch main \
  --app-location / \
  --output-location dist \
  --login-with-github

This command will create an Azure Static Web App resource with the following parameters:

  • --name‚ÄîThe name of the resource, which must be globally unique
  • --resource-group‚ÄîThe name of the resource group to contain the resource
  • --location ‚ÄîThe location of the resource
  • --source‚ÄîThe name of the GitHub repository that contains the app code
  • --branch‚ÄîThe name of the GitHub branch that contains the app code
  • --app-location‚ÄîThe location of the app code in the repository
  • --output-folder‚ÄîThe folder where the app output is generated
  • --login-with-github‚ÄîThe GitHub personal access token that grants access to the repository

The command creates a GitHub Actions workflow file in the repository that triggers the app build and deployment whenever a change is pushed to the branch. It also outputs some information about the resource, like this:

You‚Äôve now created an Azure Static Web App resource and a GitHub Actions workflow for the app.

To link the function app from part 1 as the back end for the Azure Static Web App, you use az staticwebapp backends link. This command links a pre-existing back end with a static web app, also known as ‚ÄúBring your own API.‚Äù You need to provide the function app‚Äôs resource ID, the static web app‚Äôs resource group, and the back-end region.

Link the function app as the back end for the static web app by running the following:

  --backend-resource-id "/subscriptions/<subscription-id>/resourceGroups/<resource-group>/providers/Microsoft.Web/sites/<function-app-name>" \
  --name virtual-stylist-chat \
  --resource-group <your-resource-group> \
  --backend-region westus
